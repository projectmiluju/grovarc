# 유저 플로우 설계

> Phase 0 산출물 — Grovarc의 핵심 유저 시나리오를 정의합니다.

---

## 핵심 시나리오 요약

```
[온보딩] → [일상 루프: 로그 작성] → [주간 회고 자동 생성] → [성장 코칭 요청]
```

---

## A. 온보딩 플로우

```
1. 랜딩 페이지 접속
   └─ "시작하기" CTA 클릭

2. 회원가입
   ├─ 이메일 + 비밀번호
   └─ (추후) GitHub OAuth

3. 프로필 설정
   ├─ 직군 선택 (백엔드 / 프론트엔드 / 풀스택 / 기타)
   ├─ 경력 연차
   ├─ 관심 기술 스택 (다중 선택)
   └─ 주간 목표 시간 설정

4. 첫 로그 작성 유도
   └─ 온보딩 가이드 팝업 → "오늘 한 일 기록하기" 버튼
```

---

## B. 일상 루프 — 작업 로그 작성

```
1. 대시보드 접속
   └─ 오늘 로그가 없으면 → "오늘 로그 작성" 배너 표시

2. 로그 작성 페이지
   ├─ 제목 (선택)
   ├─ 작업 내용 (자유 텍스트 + Markdown 지원)
   ├─ 태그 (기술 스택, 카테고리: 개발 / 학습 / 회의 / 리뷰 등)
   ├─ 소요 시간
   └─ 난이도 / 감정 상태 (선택, 1~5)

3. 로그 저장
   ├─ Spring Boot API → PostgreSQL 저장
   └─ Kafka 이벤트 발행 → AI 서버 비동기 전달

4. AI 즉각 피드백 (비동기, 저장 후 수 초 내)
   ├─ 오늘 로그 요약 1줄
   ├─ 연관 과거 로그 패턴 언급 (RAG)
   └─ 간단한 칭찬 or 제안 메시지

5. 대시보드 업데이트
   ├─ 이번 주 작업 시간 누적
   ├─ 기술 스택 태그 빈도
   └─ 연속 기록 스트릭 (잔디 차트)
```

---

## C. 주간 회고 플로우 (자동 + 유저 검토)

```
[자동 — Celery Beat, 매주 일요일 오후 6시]

1. AI Agent 실행
   ├─ 지난 7일 로그 수집
   ├─ LangGraph: 패턴 분석
   │   ├─ 주로 작업한 기술 스택
   │   ├─ 시간 분포 (개발 vs 학습 vs 회의)
   │   ├─ 난이도 트렌드
   │   └─ 감정 상태 변화
   ├─ Fine-tuned LLaMA 3 → 회고 초안 생성
   │   ├─ 이번 주 잘한 점 (KPT: Keep)
   │   ├─ 개선할 점 (Problem)
   │   └─ 다음 주 시도할 것 (Try)
   └─ 유저에게 알림 발송 (이메일 / 인앱)

[유저 액션]

2. 알림 수신 → 회고 페이지 접속

3. AI 초안 검토
   ├─ 섹션별 수정 가능 (Keep / Problem / Try)
   ├─ AI 제안 수용 / 거부 피드백 버튼
   └─ 추가 코멘트 직접 작성

4. 회고 저장
   ├─ PostgreSQL 저장
   └─ 피드백 데이터 → 다음 회고 학습에 반영

5. 성장 리포트 업데이트
   └─ 월간 / 분기 성장 차트 갱신
```

---

## D. 성장 코칭 플로우 (유저 요청 시)

```
1. 대시보드 → "성장 코칭 받기" 버튼 클릭

2. AI Agent 실행
   ├─ 최근 3개월 로그 분석
   ├─ RAG: 부족한 기술 스택 파악
   ├─ Web Search Tool: 관련 학습 리소스 검색
   └─ 개인화된 학습 로드맵 생성
       ├─ 단기 (2주): 추천 자료 3개
       ├─ 중기 (1개월): 토이 프로젝트 제안
       └─ 장기 (3개월): 목표 기술 스택 달성 계획

3. 로드맵 페이지 표시
   ├─ 로드맵 저장 / 공유
   └─ 각 항목 완료 체크 → 다음 로그 작성 시 연동
```

---

## E. MCP 서버 연동 플로우 (Cursor / Claude 사용 시)

```
1. 개발자가 Cursor IDE에서 MCP 서버 연결

2. 자연어 쿼리
   예) "지난 주 내가 뭘 했는지 요약해줘"
       "React 관련 로그만 보여줘"
       "이번 달 회고 초안 뽑아줘"

3. MCP 서버 → Grovarc API 호출 → 결과 반환

4. IDE 내에서 바로 로그 조회 / 회고 확인
```

## 페이지 목록 (UI 기준)

| 페이지 | 경로 | 설명 |
|--------|------|------|
| 랜딩 | `/` | 서비스 소개, CTA |
| 회원가입 | `/signup` | 이메일 가입 |
| 로그인 | `/login` | 이메일 / (추후 GitHub OAuth) |
| 대시보드 | `/dashboard` | 스트릭, 주간 요약, 최근 로그 |
| 로그 작성 | `/logs/new` | 오늘 작업 기록 |
| 로그 목록 | `/logs` | 전체 로그 히스토리 |
| 로그 상세 | `/logs/:id` | 단일 로그 + AI 피드백 |
| 주간 회고 | `/retrospectives` | 회고 목록 |
| 회고 상세 | `/retrospectives/:id` | AI 초안 검토 / 수정 |
| 성장 코칭 | `/coaching` | 로드맵 요청 / 확인 |
| 프로필 | `/profile` | 설정, 기술 스택 관리 |

## 우선순위 (MVP 기준)

**MVP에 포함**
- 온보딩 (A)
- 로그 작성 + AI 즉각 피드백 (B)
- 주간 회고 자동 생성 + 검토 (C)

**MVP 이후**
- 성장 코칭 (D)
- MCP 서버 연동 (E)
- GitHub OAuth
- 공유 기능
